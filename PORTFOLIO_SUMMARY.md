## How to Use This Project in Interviews

### Interview Talking Points:

**"Tell me about a complex project you've done?"**
> "I created a supply chain analytics project that analyzes 22,800 transactions across 5 tables. It includes 12 SQL queries ranging from basic aggregations to advanced window functions and CTEs. The project isn't just technical—it includes business analysis showing how a 14% late delivery rate impacts customer satisfaction, with specific recommendations to improve it to <5%."

**"What's an example of providing insight from data?"**
> "In this project, I identified that 5-10% of products have profit margins below 10%, dragging down overall profitability. I recommended a SKU rationalization process that could improve overall margins by 2-3%. I also quantified the customer acquisition cost opportunity from focusing on Platinum customers."

**"How do you approach SQL query optimization?"**
> "I design queries with business questions first, then build the SQL. I use CTEs to break complex logic into readable steps, avoid SELECT *, and use window functions instead of self-joins. I included several optimized queries in my project showing this progression."

**"Walk me through your data pipeline."**
> "My Python script automates loading CSV files into SQLite with validation checks. It verifies row counts, logs errors, and provides a summary report. This makes the pipeline reproducible and easy to debug—important for real-world production use."
